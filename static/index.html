系统架构

四层对话管理架构

彩虹城 AI Agent 对话管理系统采用了四层架构设计，提供清晰的数据组织和管理能力， 确保对话的连贯性、上下文理解和多模态支持。

消息层（Message）

单条消息内容，支持文本、图像等多种类型

轮次层（Turn）

单轮对话，包含用户输入和系统响应

会话层（Session）

一次完整交互，包含多个轮次

对话层（Dialogue）

最高层级，包含多个会话和对话元数据

开发接口

API 文档

彩虹城 AI Agent 提供完整的 RESTful API，方便与各种应用和服务集成

POST /api/dialogue/input
处理用户输入

接收用户输入，创建或继续会话，并返回AI响应。

请求参数：
{
  "input": "string",  // 用户输入内容
  "session_id": "string",  // 可选，会话 ID
  "dialogue_type": "human_to_ai",  // 对话类型
  "metadata": {}  // 可选，元数据
}
响应：
{
  "response": "string",  // AI 响应内容
  "session_id": "string",  // 会话 ID
  "turn_id": "string",  // 轮次 ID
  "tool_calls": []  // 工具调用结果（如有）
}
GET /api/dialogue/types
获取支持的对话类型

返回系统支持的所有对话类型。

响应：
{
  "dialogue_types": [
    "human_to_ai",
    "ai_to_ai",
    "human_to_multi_ai",
    "multi_human_to_ai",
    "multi_human_to_multi_ai",
    "moderated_human_to_ai",
    "moderated_ai_to_ai"
  ]
}
POST /api/session
创建新会话

创建新的对话会话。

请求参数：
{
  "dialogue_type": "human_to_ai",  // 对话类型
  "metadata": {}  // 可选，元数据
}
响应：
{
  "session_id": "string",  // 新创建的会话 ID
  "created_at": "2023-01-01T12:00:00Z",  // 创建时间
  "dialogue_type": "human_to_ai"  // 对话类型
}
GET /api/session/{session_id}
获取会话信息

获取指定会话的详细信息和历史记录。

响应：
{
  "session_id": "string",
  "created_at": "2023-01-01T12:00:00Z",
  "updated_at": "2023-01-01T12:05:00Z",
  "dialogue_type": "human_to_ai",
  "turns": [  // 会话中的轮次列表
    {
      "turn_id": "string",
      "created_at": "2023-01-01T12:01:00Z",
      "messages": []  // 轮次中的消息列表
    }
  ]
}
POST /api/session/{session_id}/turn
创建新轮次

在指定会话中创建新的对话轮次。

请求参数：
{
  "metadata": {}  // 可选，元数据
}
响应：
{
  "turn_id": "string",  // 新创建的轮次 ID
  "session_id": "string",  // 会话 ID
  "created_at": "2023-01-01T12:00:00Z"  // 创建时间
}
GET /api/turn/{turn_id}
获取轮次信息

获取指定轮次的详细信息和消息列表。

响应：
{
  "turn_id": "string",
  "session_id": "string",
  "created_at": "2023-01-01T12:00:00Z",
  "messages": [  // 轮次中的消息列表
    {
      "message_id": "string",
      "role": "user",
      "content": "string",
      "created_at": "2023-01-01T12:01:00Z"
    }
  ]
}
POST /api/turn/{turn_id}/message
添加消息

在指定轮次中添加新消息。

请求参数：
{
  "role": "user",  // 消息发送者角色（user 或 assistant）
  "content": "string",  // 消息内容
  "content_type": "text",  // 内容类型（text, image 等）
  "metadata": {}  // 可选，元数据
}
响应：
{
  "message_id": "string",  // 新创建的消息 ID
  "turn_id": "string",  // 轮次 ID
  "role": "user",  // 消息发送者角色
  "content": "string",  // 消息内容
  "created_at": "2023-01-01T12:00:00Z"  // 创建时间
}
GET /api/message/{message_id}
获取消息信息

获取指定消息的详细信息。

响应：
{
  "message_id": "string",
  "turn_id": "string",
  "role": "user",
  "content": "string",
  "content_type": "text",
  "created_at": "2023-01-01T12:00:00Z",
  "metadata": {}
}
